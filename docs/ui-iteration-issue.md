## 背景

当前 OmniPost V1 已定义为“三栏式多平台内容创作工作台”，核心闭环是：

```text
输入内容 → 平台适配 → 编辑确认 → 平台预览 → 模拟发布
```

截图中的 ContentFlow UI 在此基础上给出了更成熟的产品形态：左侧固定导航、顶部操作栏、主编辑区、平台预览区、排版样式工具栏、发布设置和内容检测。它比现有 V1 文档更强调“写作、预览、排版、发布”的同屏闭环，适合作为下一轮 UI 迭代目标。

## 用户故事

作为多平台内容创作者，我希望在一个页面内完成内容编辑、平台预览、样式调整、发布设置和内容检测，这样我可以在发布前快速确认内容是否适合各个平台，并减少来回切换页面的成本。

## 目标

- 将创作台升级为更完整的内容工作台：左侧编辑、中央预览、右侧排版工具、底部发布确认。
- 强化顶部状态反馈：保存状态、预览全平台、保存草稿、一键发布。
- 保留 OmniPost 的多平台适配能力，并让平台 Tab、发布设置、内容检测形成明确闭环。
- 让 UI 信息层级更稳定，适配桌面端 1280px 以上主流工作场景。

## 范围

### P0 必做

- 新增应用级壳层：左侧 Sidebar + 顶部 Header。
- 重构 `/workspace` 主页面布局，参考效果图，形成四个核心区域：
  - 内容编辑区
  - 平台预览区
  - 排版样式区
  - 发布设置区

- 平台 Tab 支持公众号、知乎、B站、小红书切换。
- 预览区继续复用现有 `PreviewRenderer` 和各平台预览组件。
- 发布设置区展示平台授权状态、发布能力状态、发布模式选择和主发布按钮。
- 保证桌面端布局不溢出、不重叠，主要操作可见。

### P1 增强

- 排版样式控件真正影响预览渲染，例如标题样式、正文样式、引用块、分割线、重点强调。
- 支持平台预览全屏或弹窗预览。
- 保存草稿按钮接入当前内容状态。
- 发布设置与现有模拟发布任务打通，发布后进入发布记录。
- 移动端降级为分段 Tab：编辑 / 预览 / 样式 / 发布。

### 暂不包含

- 真实平台授权与真实发布 API。
- 完整富文本编辑器能力。
- 复杂排版模板市场。
- AI 自动视觉设计或自动封面生成。
- 版本升级管理内容
- 发布内容检测

## 任务拆解

### 1. 应用壳层与导航

- [ ] 重构 `src/components/shell/AppNav.tsx`，支持截图风格的左侧固定导航。
- [ ] 导航项包含：内容中心、新建内容、模板管理、平台账号、发布记录、系统设置。
- [ ] 顶部 Header 增加返回入口、页面标题、保存状态、用户菜单。

验收标准：

- 左侧导航宽度固定，主内容区域随窗口自适应。
- 当前菜单高亮清晰。
- 顶部操作区在 1280px 宽度下不换行、不遮挡。

### 2. 工作台布局重构

- [ ] 重构 `src/components/workspace/WorkspaceShell.tsx` 的页面栅格。
- [ ] 第一行布局为：内容编辑区 / 平台预览区 / 排版样式区。
- [ ] 第二行布局为：发布设置区 / 内容检测区。
- [ ] 控制整体页面高度，核心工作区可滚动但顶栏稳定。

验收标准：

- 1440px 桌面视口下，五个区域均在首屏或接近首屏可见。
- 内容较长时只在对应面板内部滚动，不挤压其它区域。
- 页面没有横向滚动条。

### 3. 内容编辑区

- [ ] 将现有 `LeftPanel` 升级为内容编辑面板。
- [ ] 字段包含：标题、内容类型、标签、封面图、正文 Markdown 编辑器。
- [ ] 标签支持新增、删除和展示已有标签。
- [ ] 封面图展示缩略图和上传占位。
- [ ] Markdown 工具栏展示基础按钮：加粗、斜体、标题、列表、链接、图片、引用、代码、撤销、重做。
- [ ] 底部展示 Markdown 状态和字数。

验收标准：

- 标题、正文、标签变更可以同步到当前工作流状态。
- 字数统计随正文输入更新。
- 无图状态、已上传状态都有明确视觉。

### 4. 平台预览区

- [ ] 将现有 `CenterPanel` 迭代为平台预览主区域，复用 `PreviewRenderer`。
- [ ] 顶部增加平台 Tab：公众号、知乎、B站、小红书。
- [ ] Tab 展示平台图标、平台名称和选中状态。
- [ ] 预览区展示 AI 生成说明、文章标题、作者信息、时间、阅读量、封面图、正文。
- [ ] 预览区支持空状态、生成中状态、生成失败状态。

验收标准：

- 切换平台 Tab 时，预览内容和校验状态同步变化。
- 各平台预览有明显差异，不只是颜色变化。
- 内容超长时预览面板内部滚动。

### 5. 排版样式区

- [ ] 新增或重构 `RightPanel` 为排版样式面板。
- [ ] 支持样式选择：清新简约、专业干货、轻松种草等。
- [ ] 支持标题样式：标题、副标题、小标题。
- [ ] 支持正文样式：正文、引用、列表。
- [ ] 支持内容组件：图片、分割线、二维码。
- [ ] 支持重点强调：加粗、颜色、背景。

验收标准：

- P0 阶段控件可选中并更新 UI 状态。
- P1 阶段控件能影响预览渲染。
- 控件尺寸稳定，文字不溢出。

### 6. 发布设置区

- [ ] 新增发布设置组件。
- [ ] 平台卡片展示平台名称、授权状态、发布能力和选中状态。
- [ ] 发布模式支持：真实发布、模拟发布。
- [ ] 默认选择模拟发布。
- [ ] 主按钮为“立即发布”，未满足条件时进入禁用或提示状态。
- [ ] 支持定时发布入口，P0 可只展示 UI。

验收标准：

- 已授权和未授权平台视觉区分明显。
- 模拟发布能复用现有 `POST /api/publish/mock`。
- 发布成功后能在发布记录中查看任务。

## 验收清单

- [ ] `/workspace` 首屏信息层级与截图一致：导航、顶栏、编辑、预览、样式、发布、检测。
- [ ] 用户可以完成：输入内容 → 切换平台预览 → 调整样式 → 检测 → 模拟发布。
- [ ] 1280px、1440px、1920px 桌面视口无布局重叠。
- [ ] 主要交互均有 hover、active、disabled、loading 状态。
- [ ] 现有 API、validators、preview 组件未被破坏。
- [ ] `npm run typecheck` 通过。
- [ ] `npm run build` 通过。

## 风险

- 信息密度较高，低分辨率下容易拥挤，需要优先保证 1280px 桌面体验。
- 排版样式如果一次性做成真实编辑器，范围会快速膨胀；建议 P0 先做 UI 状态，P1 再打通渲染。
- 真实发布与账号授权容易拖慢迭代节奏，本轮以模拟发布和授权状态展示为主。
