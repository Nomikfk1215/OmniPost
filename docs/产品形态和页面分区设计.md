# OmniPost V1 产品形态和页面分区设计

> 文档定位：页面级产品设计  
> 关联文档：`prd.md`、`渠道分析.md`  
> 目标：让开发和评审能看懂产品具体长什么样，每个页面有哪些区块，用户如何操作。

---

## 1. 产品形态总览

OmniPost V1 的产品形态是：

> **三栏式多平台内容创作工作台**

核心界面不是聊天框，也不是传统后台表格，而是一个内容编辑器。

```text
左边：原始内容输入
中间：各平台版本编辑
右边：当前平台预览和规则校验
```

用户在一个主工作台内完成主要操作：

```text
输入
  ↓
生成
  ↓
编辑
  ↓
预览
  ↓
模拟发布
```


## 3. V1 页面优先级

| 页面 | 优先级 | 是否必做 | 说明 |
|---|---|---|---|
| 创作台 | P0 | 是 | 核心页面，承载输入、生成、编辑、预览、校验、模拟发布 |
| 发布记录页 | P0 | 是 | 查看历史模拟发布任务 |
| 工作台首页 | P1 | 可合并 | 只作为入口，时间紧可直接进入创作台 |
| 通用模拟详情页 | P1 | 可选 | 复用预览组件展示模拟链接 |

## 5. 页面一：工作台首页

### 5.1 页面目标

让用户明确产品用途，并快速开始一个多平台发布任务。

### 5.2 页面布局

```text
┌──────────────────────────────────────────────────────────────┐
│ 顶部导航                                                      │
├──────────────────────────────────────────────────────────────┤
│ Banner 区                                                     │
│ 一次输入，自动生成公众号 / 知乎 / 小红书 / B站专栏版本          │
│ [ 新建多平台内容 ] [ 查看发布记录 ]                            │
├──────────────────────────────────────────────────────────────┤
│ 能力卡片区                                                    │
│ [文字输入] [多平台适配] [预览校验] [模拟发布]                  │
├──────────────────────────────────────────────────────────────┤
│ 最近任务区                                                    │
│ 标题 | 平台 | 状态 | 更新时间 | 操作                           │
└──────────────────────────────────────────────────────────────┘
```

### 5.3 Banner 区

字段：

```text
主标题：一次创作，多平台适配
副标题：输入一份内容，自动生成公众号、知乎、小红书、B站专栏版本
主按钮：新建多平台内容
次按钮：查看发布记录
```

### 5.4 能力卡片区

四张卡片：

| 卡片 | 文案 |
|---|---|
| 文字输入 | 支持粘贴文章、输入脚本、上传配图 |
| 多平台适配 | 自动生成不同平台的标题、正文、标签 |
| 预览校验 | 在右侧查看平台样式和规则提醒 |
| 模拟发布 | 一键生成模拟发布链接和发布记录 |

### 5.5 最近任务区

表格字段：

| 字段 | 说明 |
|---|---|
| 标题 | 原始内容标题 |
| 目标平台 | 已选择的平台 |
| 状态 | 草稿 / 已生成 / 已模拟发布 |
| 更新时间 | 最近修改时间 |
| 操作 | 继续编辑 / 查看模拟结果 |

## 6. 页面二：创作台

创作台是 V1 最核心页面。

### 6.1 创作台整体布局

```text
┌──────────────────────────────────────────────────────────────┐
│ 顶部导航                                                      │
├──────────────────────────────────────────────────────────────┤
│ 步骤条：1 输入内容 → 2 平台适配 → 3 预览确认 → 4 模拟发布       │
├──────────────────────────────────────────────────────────────┤
│ 发布目标： [✓公众号] [✓知乎] [✓小红书] [✓B站专栏]              │
│ 风格预设： (●) 专业干货型   ( ) 轻松种草型                    │
│ 操作按钮： [生成平台版本] [保存草稿]                           │
├──────────────────────────────────────────────────────────────┤
│ 左栏：原始内容输入      │ 中栏：平台版本编辑     │ 右栏：预览与校验 │
│                         │                       │                │
│                         │                       │                │
└──────────────────────────────────────────────────────────────┘
```

### 6.2 三栏宽度建议

| 区域 | 宽度 | 作用 |
|---|---|---|
| 左栏 | 30% | 原始输入 |
| 中栏 | 40% | 平台内容编辑 |
| 右栏 | 30% | 预览和校验 |

如果屏幕宽度不够，可改为：

```text
左侧输入 + 右侧 Tab
```

但 Demo 推荐三栏，更直观。

## 10. 左栏模式一：文字 + 图片

### 10.1 适用场景

用户已经有：

```text

笔记
视频脚本

```

### 10.2 字段

| 字段 | 类型 | 是否必填 | 说明 |
|---|---|---|---|
| 标题 | input | 否 | 为空时 AI 自动生成 |
| 正文 | textarea / markdown editor | 是 | 核心原始内容 |
| 图片 | upload | 否 | 可上传多张 |
| 用户标签 | tag input | 否 | 可作为 AI 参考 |

### 10.3 交互

```text
用户输入正文
  ↓

  ↓
选择平台和风格
  ↓

```

## 15. 右栏：规则校验区

### 15.1 校验原则

校验由代码完成，不交给 AI 自己判断。

### 15.2 校验等级

```text
通过：绿色
警告：黄色
失败：红色
```

### 15.3 各平台校验示例

#### 公众号

```text
✅ 标题不为空
✅ 摘要 80 字，符合 120 字以内
✅ 正文不为空
⚠ 未上传封面图，建议补充
```

#### 知乎

```text
✅ 标题为问题式
✅ 正文包含分点结构
⚠ 标签数量较少，建议 2～5 个
```

#### 小红书

```text
✅ 标题 8 字，符合短标题要求
✅ 标签 5 个，符合 3～8 个要求
✅ 正文包含 emoji 和分点
⚠ 当前没有 3:4 封面图
```

#### B站专栏

```text
✅ 标题不为空
✅ 简介已生成
✅ 标签 4 个
✅ 已生成分区建议：知识
```

## 20. 页面设计总结

OmniPost V1 的用户感知重点是：

```text
我只写了一份内容
系统真的帮我生成了不同平台的版本
我能编辑这些版本
我能看到它们发出去大概长什么样
我能一键完成模拟发布
```

产品形态应该始终围绕：

```text
左栏原文
中栏多平台版本
右栏预览校验
```
