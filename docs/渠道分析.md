# OmniPost V1 渠道分析

> 文档定位：平台/渠道差异分析  
> 关联文档：`prd.md`、`产品形态和页面分区设计.md`  
> 目标：明确 V1 首批四个平台的内容差异、格式要求、发布流程差异和案例输出，避免平台适配只停留在“改语气”。
> 范围说明：本文件只提供平台规则和内容案例，不定义开发范围；开发范围以 `prd.md` 第 5 节为准。

---

## 1. 渠道选择

V1 首批选择四个平台：

```text
微信公众号
知乎
小红书
B站专栏
```

选择原因：

| 平台 | 选择原因 |
|---|---|
| 微信公众号 | 长文代表平台，适合展示长文排版和摘要能力 |
| 知乎 | 知识问答代表平台，适合展示逻辑改写能力 |
| 小红书 | 图文种草代表平台，适合展示短内容、标签、emoji 和图片建议 |
| B站专栏 | 年轻化知识内容平台，适合展示标题、简介、标签、分区能力 |

## 2. 四个平台核心差异总览

| 维度 | 微信公众号 | 知乎 | 小红书 | B站专栏 |
|---|---|---|---|---|
| 内容载体 | 长图文 | 回答 / 文章 | 图文笔记 | 专栏 / 视频补充 |
| 用户心智 | 深度阅读、私域沉淀 | 找答案、看经验 | 种草、收藏、生活方式 | 年轻化知识分享 |
| 标题风格 | 信息明确，有传播感 | 问题式、观点式 | 短、强结果、情绪化 | 有看点、年轻化 |
| 正文结构 | 引言 + 分节 + 总结 | 结论先行 + 分点论证 | 痛点 + 体验 + 清单 + 标签 | 简介 + 分点正文 |
| 语气 | 稳重、完整 | 理性、可信 | 口语、轻快、有情绪 | 轻松、有梗但有信息量 |
| 图片作用 | 封面和正文辅助 | 辅助说明 | 核心内容载体 | 封面和辅助说明 |
| 标签作用 | 弱 | 中 | 强 | 较强 |
| V1 输出重点 | 标题、摘要、HTML 正文、封面建议 | 问题式标题、结论、正文、标签 | 短标题、emoji、短句、话题标签、图片建议 | 标题、简介、正文、标签、分区建议 |
| V1 发布方式 | 模拟发布 | 模拟发布 / 复制发布包 | 模拟发布 | 模拟发布 |

## 3. 微信公众号渠道分析

### 3.1 内容特点

微信公众号适合：

```text
长文
结构化表达
观点沉淀
私域传播
排版阅读
```

它不适合只生成短句或碎片化内容。公众号版本要更像一篇完整文章。

### 3.2 典型结构

```text
标题
摘要
封面
引言
一级小标题
正文段落
重点加粗 / 引用
总结
文末引导
```

### 3.3 标题特点

适合：

```text
清晰
有信息量
能表达收益
适合转发
```

示例：

```text
如何用 AI 提升学习效率？一套普通人也能落地的学习工作流
```

不适合：

```text
太短
只有情绪
标题党严重
大量 emoji
```

### 3.4 正文特点

正文应具备：

```text
段落完整
小标题清晰
逻辑递进
适合阅读
可转 HTML / 富文本
```

推荐结构：

```text
开头提出问题
  ↓
分析为什么低效
  ↓
给出方法步骤
  ↓
举例说明
  ↓
总结方法论
```

### 3.5 图片要求

V1 不做真实图片裁剪，但要给出建议：

```text
是否需要封面
封面关键词
正文配图位置建议
```

### 3.6 V1 输出字段

```json
{
  "platform": "wechat",
  "title": "string",
  "digest": "string",
  "html": "string",
  "coverSuggestion": "string",
  "imageSuggestions": ["string"]
}
```

### 3.7 校验规则

```text
标题不能为空
摘要建议不超过 120 字
正文不能为空
建议有封面图或封面建议
正文应包含分段或小标题
```

### 3.8 V1 发布流程

```text
生成公众号版本
  ↓
公众号样式预览
  ↓
模拟发布
```

### 3.9 V2 真实发布流程参考

如果接真实发布，可以考虑：

```text
获取 access_token
  ↓
上传封面 / 正文图片素材
  ↓
新建草稿
  ↓
提交发布
  ↓
查询发布状态
```

V1 不依赖该流程。

## 4. 知乎渠道分析

### 4.1 内容特点

知乎适合：

```text
问题式内容
经验分享
理性分析
专业回答
个人观点
```

知乎版本不应该像小红书那样大量使用 emoji，也不应该过度营销。

### 4.2 典型结构

```text
问题式标题
开头直接给结论
背景解释
分点论证
方法建议
注意事项
总结
```

### 4.3 标题特点

适合：

```text
普通人如何用 AI 提升学习效率？
AI 真的能提升学习效率吗？
如何用 AI 制定一套可执行的学习计划？
```

不适合：

```text
AI学习法真的香！
姐妹们快试试！
不看后悔！
```

### 4.4 正文特点

正文应具备：

```text
结论先行
分点清楚
理由充分
减少夸张表达
有个人经验或方法论
```

### 4.5 图片要求

图片不是核心，但可以作为辅助说明：

```text
流程图
对比图
笔记截图
案例截图
```

### 4.6 V1 输出字段

```json
{
  "platform": "zhihu",
  "title": "string",
  "openingConclusion": "string",
  "body": "string",
  "tags": ["string"]
}
```

### 4.7 校验规则

```text
标题建议是问题式或观点式
正文不能为空
正文建议包含分点结构
标签建议 2～5 个
避免大量 emoji
避免强营销语气
```

### 4.8 V1 发布流程

```text
生成知乎版本
  ↓
知乎样式预览
  ↓
模拟发布 / 复制发布包
```

V1 不承诺知乎官方 API 一键发布。

## 5. 小红书渠道分析

### 5.1 内容特点

小红书适合：

```text
图文笔记
种草
经验分享
收藏型内容
生活方式表达
强视觉内容
```

小红书的核心不是“短”，而是：

```text
短标题
图片优先
口语化
强结果
emoji
分点
话题标签
```

### 5.2 典型结构

```text
短标题
痛点开头
个人体验
清单方法
结果反馈
适合人群
话题标签
```

### 5.3 标题特点

适合：

```text
AI学习法真的香！
这套学习流程救了我
普通人也能用的 AI 学习法
```

不适合：

```text
如何用 AI 提升学习效率？一套普通人也能落地的学习工作流
```

这个标题更像公众号，不像小红书。

### 5.4 正文特点

正文应具备：

```text
短句
换行多
emoji
分点
语气口语化
有收藏价值
```

示例结构：

```text
以前学习总是没计划，
资料越收藏越焦虑😭

后来我试了一个 AI 学习流程👇

✅ 先让 AI 拆学习目标
✅ 再让 AI 整理资料重点
✅ 每天生成一个学习计划
✅ 学完后自动生成复盘问题

适合：
✨ 学习没计划的人
✨ 资料太多看不完的人
✨ 想提高效率的人

#AI学习 #效率工具 #学习方法
```

### 5.5 图片要求

小红书图片非常重要。

V1 至少要给出：

```text
封面建议
图片比例建议
主图关键词
长图是否可作为主图
```

建议：

```text
3:4 封面
封面大字突出关键词
步骤类内容适合做卡片图
```

### 5.6 长图处理

小红书支持两种方式：

#### 方式一：识别改写型

```text
长图 → 识别文字 → 改写成小红书笔记
```

#### 方式二：原图搬运型

```text
长图作为主图
  ↓
AI 生成标题、正文和标签
```

V1 可以支持方式二，但只建议作为小红书等图文平台的特殊模式。

### 5.7 V1 输出字段

```json
{
  "platform": "xiaohongshu",
  "title": "string",
  "body": "string",
  "tags": ["string"],
  "imageSuggestions": ["string"],
  "interactionGuide": "string"
}
```

### 5.8 校验规则

```text
标题建议较短
正文应有短句和分点
标签建议 3～8 个
标签建议带 #
建议有图片或图片建议
如果是专业干货型，可以减少 emoji，但不能完全失去小红书结构
```

### 5.9 V1 发布流程

```text
生成小红书版本
  ↓
小红书手机卡片预览
  ↓
模拟发布
```

## 6. B站专栏渠道分析

### 6.1 内容特点

B站专栏适合：

```text
年轻化知识分享
视频内容补充
教程说明
观点长文
系列内容
```

B站专栏不是纯短内容，也不是严肃论文风。它介于公众号和小红书之间：

```text
可以轻松
但要有信息量
标题要有看点
标签和分区有价值
```

### 6.2 典型结构

```text
标题
简介
正文开头
分点内容
总结
标签
分区建议
```

### 6.3 标题特点

适合：

```text
我用 AI 搭了一套学习工作流，效率确实变高了
AI 不是替你学习，而是帮你把学习流程变清楚
```

不适合：

```text
如何用 AI 提升学习效率？一套普通人也能落地的学习工作流
```

该标题可用，但偏公众号，B站可更口语和更有看点。

### 6.4 正文特点

正文应具备：

```text
开头轻松
表达不僵硬
可以分点
可以像视频文案延展
适合年轻用户阅读
```

### 6.5 图片要求

V1 给出：

```text
封面建议
正文图建议
如果内容来自视频，可提示生成视频简介
```

## 7. 同一内容的四平台案例

### 7.1 原始内容

```text
主题：如何用 AI 提升学习效率

很多人在学习时没有计划，也不知道如何复盘。
可以用 AI 帮助自己拆解学习目标、整理资料、生成每日学习计划，并在学习后生成复盘问题。
这样可以提升学习效率，也能更快发现知识漏洞。
```

### 7.2 微信公众号版本

```text
标题：
如何用 AI 提升学习效率？一套普通人也能落地的学习工作流

摘要：
本文从目标拆解、资料整理、学习计划和复盘四个环节，介绍一套适合普通人的 AI 学习方法。

正文：
很多人在学习时遇到的最大问题，不是没有资料，而是缺少清晰的学习路径。

一、先用 AI 拆解学习目标
在开始学习之前，可以先告诉 AI：我想在两周内掌握某个知识点，请帮我拆解学习路径。

二、用 AI 整理资料
面对大量资料时，可以让 AI 帮你归纳核心概念、重点章节和学习顺序。

三、生成每日学习计划
AI 可以根据你的可用时间，生成每天的学习任务，降低执行难度。

四、用 AI 做复盘
学习完成后，让 AI 根据你的笔记生成复盘问题，帮助你发现知识漏洞。

总结：
AI 不应该替代学习，而应该成为学习过程中的辅助系统。
```

### 7.3 知乎版本

```text
标题：
普通人如何用 AI 提升学习效率？

正文：
先说结论：AI 最适合用来辅助学习流程，而不是直接替你学习。

我建议把 AI 放在四个环节里使用。

第一，拆解学习目标。
很多人学习效率低，是因为目标太大。例如“学会 Python”并不是一个好目标。你可以让 AI 帮你拆成语法基础、项目练习、资料阅读和复盘测试几个阶段。

第二，整理资料。
当你面对大量教程和文章时，可以让 AI 帮你总结核心概念，但不要完全依赖它的结论。

第三，生成学习计划。
你可以告诉 AI 自己每天可用的时间，让它给出更现实的学习安排。

第四，做复盘。
这是最容易被忽略的一步。学完之后，可以让 AI 根据笔记生成自测题，帮助你检查理解程度。

需要注意的是，AI 可以提升学习流程的效率，但不能替代你自己的思考。
```

```text
标题：
我用 AI 搭了一套学习工作流，效率确实变高了

简介：
这篇专栏分享一套适合普通人的 AI 学习流程，包括目标拆解、资料整理、学习计划和复盘方法。
```

## 9. 四个平台发布流程差异

### 9.1 V1 发布流程

| 平台 | V1 发布方式 | 说明 |
|---|---|---|
| 微信公众号 | 模拟发布 | 生成公众号文章页 |
| 知乎 | 模拟发布 / 复制发布包 | 不承诺官方 API |
| 小红书 | 模拟发布 | 生成小红书笔记页 |
| B站专栏 | 模拟发布 | 生成专栏页 |

### 9.2 V2 发布方向

| 平台 | V2 方向 | 难度 |
|---|---|---|
| 微信公众号 | 官方接口：素材、草稿、发布、状态查询 | 中 |
| B站 | 开放平台：账号授权、稿件、专栏能力 | 中到高 |
| 小红书 | 视开放平台类目和权限而定 | 高 |
| 知乎 | 发布包 / 半自动发布更稳 | 高 |

## 10. 对 PRD 和页面设计的影响

渠道分析会直接影响产品设计。

### 10.1 对字段的影响

| 平台 | 页面上必须有的字段 |
|---|---|
| 公众号 | 标题、摘要、正文、封面建议 |
| 知乎 | 问题式标题、开头结论、正文、标签 |
| 小红书 | 短标题、正文、标签、图片建议、互动引导 |
| B站专栏 | 标题、简介、正文、标签、分区建议 |

### 10.2 对校验的影响

不同平台不能用同一套校验。

例如：

```text
小红书需要校验标签数量和图片建议
公众号需要校验摘要和封面建议
知乎需要校验是否过度营销
B站需要校验简介和分区建议
```

### 10.3 对预览的影响

每个平台必须有不同预览组件：

```text
公众号：文章阅读样式
知乎：问答 / 回答样式
小红书：手机笔记卡片
B站：专栏文章样式
```

## 11. 参考资料

> 说明：V1 不依赖真实发布 API，以下资料仅用于判断 V2 真实发布扩展方向。

- B站开放平台文档显示其开放能力覆盖账号授权、视频稿件管理和专栏管理等方向。
- 小红书开放平台文档更偏应用类目、商家/服务市场、商品、素材中心等场景，个人创作者笔记发布不建议作为 V1 强依赖。
- 微信公众号真实发布一般涉及 access_token、素材上传、新建草稿、提交发布、状态查询等步骤。
- 知乎不建议在 V1 承诺官方 API 发文能力，V1 应采用模拟发布或复制发布包。
